# Orac Server API

This document describes the API for the Orac server. The server provides an interface to:
* Discover media via TMDb.
* Manage and sync a user's Trakt.tv account (lists, collections, watched status).
* Retrieve enriched media data from a local database cache.
* Trigger background processing tasks.
* Find sources for media using external scrapers.

## Group General

### Ping [GET /ping]
+ Response 200 (text/plain)
    Pong!

## Group Discover

Endpoints for discovering new media from The Movie Database (TMDb).

### Discover Media [GET /discover/{media_type}]

Discovers movies or TV shows based on filters provided as query parameters. This endpoint acts as a proxy to the TMDb discover API, processes the parameters, and formats the response. The server automatically paginates through TMDb results to return up to 100 items.

+ Parameters
    + item_type: `movie` (string, required) - The type of media to discover. Can be `movie` or `tv` (or `tvshow`).
    + ... (object) - Any valid query parameter for the TMDb discover API can be used.
        + `sort_by` (string, optional) - The sorting order for the results. Example: `popularity.desc`
        + `with_genres` (string, optional) - A comma-separated list of TMDb genre IDs to include.
        + `primary_release_date.gte` (string, optional) - Filter by release date (greater than or equal to). A special format `T-x` is supported, where `x` is the number of days in the past (e.g., `T-7` for 7 days ago).
        + `page` (number, optional) - The page number to retrieve. The server will paginate automatically to fetch up to 100 results, so this is handled internally.

+ Response 200 (application/json)

    A JSON array of discovered media items, formatted for the client. The array will contain up to 100 items.

    + Body (Example for `tv`)

        ```json
        [
            {
                "tmdb_id": 2316,
                "title": "The Office",
                "original_title": "The Office",
                "year": 2005,
                "premiered": "2005-03-24",
                "overview": "The everyday lives of office employees in the Dunder Mifflin Paper Company, Scranton, Pennsylvania branch.",
                "poster_path": "https://image.tmdb.org/t/p/w500/qj3m2u0RkppBC6sSg2a4s4S2Oa.jpg",
                "fanart_path": "https://image.tmdb.org/t/p/w780/r1jA2n2K7g6U8g1m52gS2lA210.jpg",
                "rating": 8.5,
                "votes": 1234
            }
        ]
        ```

+ Response 400 (text/plain)

    Returned if the `item_type` is not supported.

    + Body (text/plain)

        ```
        Unsupported item_type for discover: {item_type}
        ```

+ Response 500 (text/plain)

    Returned if there is an internal server error during the discover process.

    + Body (text/plain)

        ```
        Internal server error during discover
        ```

### Get Genres [GET /get_genres]

Retrieves a list of all available genres for a given media type from the local database.

+ Parameters
    + item_type: `movie` (string, required) - The type of media to get genres for. Can be `movie` or `tvshow`.

+ Response 200 (application/json)

    A JSON array of genre names.

    + Body

        ```json
        [
            "Action",
            "Adventure",
            "Animation",
            "Comedy"
        ]
        ```

+ Response 400 (text/plain)

    Returned if `item_type` is missing.

    + Body

        ```
        Missing item_type
        ```

### Get Show Details [GET /show]

Retrieves enriched details for a single TV show, including all seasons and episodes with their watched status for the current user. If the show is not in the local database, it will be fetched from TMDb.

+ Parameters
    + tmdb_id: `1399` (number, required) - The TMDb ID of the show.

+ Response 200 (application/json)

    A JSON object containing the show's full details.

    + Body (Structure Example)

        ```json

            "trakt_id": 1390,
            "title": "Game of Thrones",
            "year": 2011,
            "tmdb_id": 1399,
            "imdb_id": "tt0944947",

            "poster_path": "...",
            "fanart_path": "...",

            "seasons": [
                {
                    "season_id": "1390:S1",
                    "show_id": 1390,
                    "season": 1,
                    "title": "Season 1",
                    "overview": "Trouble is brewing in the Seven Kingdoms of Westeros...",
                    "episode_count": 10,
                    "poster_path": "...",
                    "episodes": [
                        {
                            "trakt_id": 73640,
                            "show_id": 1390,
                            "season": 1,
                            "number": 1,
                            "episode_title": "Winter Is Coming",
                            "episode_overview": "...",
                            "tmdb_id": 63056,
                            "watched_at": "2023-10-27T10:00:00.000Z",
                            "percent_watched": 100
                        }
                    ]
                }
            ]
        }
        ```

+ Response 400 (text/plain)

    Returned if `tmdb_id` is missing.

+ Response 401 (text/plain)

    Returned if the user is not authenticated with Trakt.

+ Response 404 (text/plain)

    Returned if the show is not found.

## Group Lists

Endpoints for retrieving lists and their contents.

### Get Available Lists [GET /lists]

Retrieves a list of all available "My Lists" or "Generic Lists" (from Trakt) from the local database.

+ Parameters
    + name: `my_lists` (string, optional) - The type of lists to retrieve. Can be `my_lists` (default) or `generic_lists`.
    + item_type: `All` (string, optional) - Filter lists by the type of media they contain. Can be `movie`, `tvshow`, or `All` (default).

+ Response 200 (application/json)

    A JSON array of list metadata objects.

    + Body

        ```json
        [
            {
                "name": "Watchlist",
                "user": "username",
                "slug": "watchlist",
                "item_count": 25
            },
            {
                "name": "Sci-Fi Classics",
                "user": "username",
                "slug": "sci-fi-classics",
                "item_count": 50
            }
        ]
        ```

### Get List Contents [GET /list]

Retrieves the full content of a specific list from the local database.

+ Parameters
    + name: `watchlist` (string, required) - The slug of the list.
    + item_type: `movie` (string, required) - The type of media to retrieve from the list. Can be `movie` or `tvshow`.
    + user: `username` (string, required) - The user who owns the list.

+ Response 200 (application/json)

    A JSON array of media items from the list. The structure matches the response from `/movie` or `/show` details.

## Group Trakt & Sync

Endpoints for interacting with the user's Trakt.tv account. These require the user to be authenticated with Trakt.

### Clear Trakt Collection [GET /clear_collection]

Removes all movies and shows from the user's Trakt.tv collection. This is a destructive action and requires no request body.

+ Request

    (No body)

+ Response 200 (text/plain)

    Indicates the operation was successful.

    + Body

        ```
        Collection cleared
        ```

+ Response 500 (text/plain)

    Returned if the operation fails.

    + Body

        ```
        Failed to clear collection
        ```

### Get Next Episodes [GET /next_episodes]

Retrieves a list of the next episodes to watch for the authenticated user. This is based on their watch history and excludes shows they have marked as "dropped".

+ Response 200 (application/json)

    A JSON array of enriched episode objects.

    + Body (Example)

        ```json
        [
            {
                "episode_trakt_id": 73641,
                "show_title": "Game of Thrones",
                "show_year": 2011,
                "show_overview": "Seven noble families fight for control of the mythical land of Westeros...",
                "show_trakt_id": 1390,
                "show_tmdb_id": 1399,
                "season": 1,
                "number": 2,
                "episode_title": "The Kingsroad",
                "episode_overview": "The Lannisters plot to ensure Bran's silence...",
                "air_date": "2011-04-24",
                "runtime": 56,
                "last_watched_at": null
            }
        ]
        ```

### Mark Episode Watched [PUT /watched]

Marks an episode as watched (or partially watched) and updates the user's "next episode" list. This triggers a background task to sync the watched status to Trakt.

+ Parameters
    + type: `episode` (string, required) - The type of item being marked.
    + tmdb_id: `1399` (number, required) - The TMDb ID of the **show**.
    + show_trakt_id: `1390` (number, required) - The Trakt ID of the **show**.
    + season: `1` (number, required) - The season number of the episode.
    + episode: `1` (number, required) - The episode number.
    + percent_watched: `100` (number, optional) - The percentage of the episode watched. Defaults to 100.

+ Response 200 (text/plain)

    Indicates the request was processed.

    + Body

        ```
        Episode processed.
        ```

## Group Background Tasks

Endpoints for triggering background processing and synchronization.

### Load Episodes for a Season [GET /loadepisodes]

Queues a background task to fetch and store all episode details for a specific season of a show. This is useful for pre-caching data.

+ Parameters
    + trakt_id: `1390` (number, required) - The Trakt ID of the show.
    + season: `1` (number, required) - The season number to load.

+ Response 200

    Indicates the task was successfully queued. (No response body).

## Group Scrapers

### Scrape for Sources [GET /scrape]

Finds streaming/download sources for a given movie or episode using configured external scrapers.

+ Parameters
    + tmdb_id: `603` (number, required) - The TMDb ID of the movie or show.
    + item_type: `movie` (string, required) - The type of media. Can be `movie` or `episode`.

+ Response 200 (application/json)

    A JSON object containing the scraper results.

    + Body

        ```json
        {
            "success": true,
            "provider": "limetorrents",
            "results": [
                {
                    "name": "The.Matrix.1999.1080p.BluRay.x264-...",
                    "size": "8.73 GB",
                    "seeds": 1234,
                    "url": "magnet:?xt=urn:btih:..."
                }
            ],
            "count": 1
        }
        ```
